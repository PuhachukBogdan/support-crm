# Zendesk Capture Guide — что снять для репликации и миграции

> ⚠️ **АРХИВ / выполнено.** Захват Zendesk сделан, выводы — в `../zendesk-findings.md`. Остаток «что снять ещё» см. в findings (раздел «What to capture next»).


> Цель: снять текущую конфигурацию Zendesk (скрины) и выгрузить данные (экспорт), чтобы мы **точно повторили настройки** в новой CRM и спланировали **миграцию данных**. Ты — админ, так что доступ ко всему есть.
>
> Как снимать: для **настроек** — скриншоты экранов (целиком, чтобы были видны все поля/условия); для **данных** — CSV/JSON-экспорт. Пути указаны для **Admin Center** (шестерёнка / admin.zendesk.com). Названия пунктов могут чуть отличаться по версии — ориентируйся по смыслу.
>
> Складывай всё в папку (например `zendesk-dump/`) с понятными именами: `01-fields.png`, `04-triggers-1.png`, `tickets-export.csv` и т.д.

## Как пользоваться
Иди по разделам сверху вниз. Каждый блок: **[Что]** — **[Где]** — **[Как снять]** — **[Зачем нам]**. Раздел «⭐ приоритет» снять в первую очередь — он разблокирует больше всего решений.

---

## ⭐ 1. Ticket fields (кастомные поля тикета)
- **Где:** Admin Center → Objects and rules → Tickets → Fields.
- **Как снять:** скрин списка всех полей; затем открой **каждое кастомное поле** и сними его настройки (тип, название, обязательность, значения для выпадающих списков).
- **Зачем:** это наши **custom attributes** (0021/0006). Нужен полный список полей и их типов, чтобы повторить и заложить в схему.

## ⭐ 2. Tags (теги)
- **Где:** Admin Center → Objects and rules → Tickets → Tags (список используемых тегов). Плюс где теги реально применяются — видно в триггерах/макросах.
- **Как снять:** скрин списка тегов; если есть счётчик использования — тоже.
- **Зачем:** это наши **labels** (0021). Переносим существующие + добавляем новые (тип канала и пр.) под аналитику.

## ⭐ 3. Macros (макросы)
- **Где:** Admin Center → Workspaces → Agent tools → Macros (или Workspaces → Macros).
- **Как снять:** скрин списка; открой **каждый макрос** и сними его действия (что он ставит/меняет, текст ответа).
- **Зачем:** наши **macros** (0021). Хотел «практичные макросы» — вот их реальный список как основа.

## ⭐ 4. Triggers (триггеры) и Automations (автоматизации)
- **Где:** Admin Center → Objects and rules → Business rules → **Triggers**; и там же → **Automations**.
- **Как снять:** скрин списка (с порядком!); открой каждый и сними **условия (ALL/ANY)** и **действия**. Порядок триггеров важен.
- **Зачем:** наш **движок автоматизаций** (0021). Условия+действия = правила, которые повторяем.

## ⭐ 5. Ticket identifier / пример тикета
- **Где:** любой открытый тикет в агентском интерфейсе.
- **Как снять:** скрин заголовка тикета (виден номер/ID, дата, канал, тема); 2–3 разных примера.
- **Зачем:** формат **идентификатора тикета** и терминология (0021). Ты предполагал «id-дата-канал» — сверим по факту.

---

## 6. Groups (группы) и Roles (роли)
- **Где:** Admin Center → People → Team → **Groups**; и → **Roles**.
- **Как снять:** скрин списка групп; скрин ролей — и **каждой кастомной роли** её права (галочки доступа).
- **Зачем:** группы → наши **teams** (0006); роли → наша **RBAC** (0004). Права ролей — прямой чертёж матрицы доступа.

## 7. Views (представления)
- **Где:** Admin Center → Workspaces → Agent tools → Views.
- **Как снять:** скрин списка; открой каждое активное представление — условия и колонки.
- **Зачем:** наши **custom views / фильтры**. Показывает, как команда реально сортирует работу.

## 8. SLA policies (соглашения об уровне сервиса)
- **Где:** Admin Center → Objects and rules → Business rules → Service level agreements.
- **Как снять:** скрин каждой политики SLA (цели по времени, к каким тикетам применяется).
- **Зачем:** если строим **SLA** (0012 — опционально). Даже если позже — важно знать целевые времена.
- *Примечание:* доступно не на всех тарифах — если раздела нет, просто отметь это.

## 9. Ticket forms (формы тикета)
- **Где:** Admin Center → Objects and rules → Tickets → Forms.
- **Как снять:** скрин списка форм; каждую форму — какие поля в ней и порядок.
- **Зачем:** как поля группируются для разных типов обращений.

## 10. Routing / assignment (маршрутизация/назначение)
- **Где:** Admin Center → Objects and rules → **Omnichannel routing** (если включено); и/или настройки автоназначения в группах/триггерах.
- **Как снять:** скрин настроек маршрутизации (round-robin, по нагрузке, по навыкам, лимиты).
- **Зачем:** наше **авто-назначение** (0021) — «оставить как в Zendesk». Нужна их логика распределения.

## 11. Channels (каналы) и Schedules (расписание)
- **Где:** Admin Center → **Channels** (какие подключены: email, чат, мессенджеры, соцсети); Admin Center → Objects and rules → Business rules → **Schedules** (рабочие часы).
- **Как снять:** скрин списка активных каналов; скрин расписаний/бизнес-часов.
- **Зачем:** сверить наш **channel-план** (0006) с реальностью; расписания влияют на SLA и авто-ответы.

## 12. User & Organization fields (поля пользователей/компаний)
- **Где:** Admin Center → Objects and rules → People → (User fields / Organization fields).
- **Как снять:** скрины списков кастомных полей контактов и организаций.
- **Зачем:** кастом-атрибуты **контактов/компаний** — тоже переносим.

---

## 📦 13. ЭКСПОРТ ДАННЫХ (для миграции)
- **Где:** Admin Center → Account → Tools → **Reports / Export** (или «Export»).
- **Что выгрузить (CSV/JSON, что доступно):**
  - **Tickets** (со всеми полями, комментариями если можно) — история обращений.
  - **Users / End users** (контакты).
  - **Organizations** (компании).
- **Как:** запусти экспорт, дождись письма со ссылкой, скачай файлы. **Не пересылай эти файлы никуда лишнему** — в них персданные клиентов.
- **Зачем:** это исходные данные для **миграции** (0026 §6) в нашу схему.
- *Примечание:* полный экспорт доступен обычно на Professional+ тарифах. Если недоступен — отметь, тогда пойдём через **Zendesk API** (см. ниже).

## 🔌 14. (Опционально) Через API
Если что-то не выгружается из UI — Zendesk REST API (`/api/v2/...`) отдаёт: `ticket_fields`, `triggers`, `automations`, `macros`, `views`, `groups`, `custom_roles`, `sla_policies`, `tickets`, `users`, `organizations`. Скажи — распишу конкретные запросы под твой сабдомен и токен.
